About AIQX Music Catalog App

Objective

Develop an Angular application that presents a visually appealing music album catalog.

Functional Requirements

Album Display

  • Present albums as cards with hover tooltips displaying the album name.
  • Clicking an album card navigates to a dedicated album details page.

Header Menu

  • Implement a header menu with two options: "Home" and "About."
  • "Home" displays the album catalog.
  • "About" provides information about the application.

Data Consumption (albums.json)

Consume album data from a local albums.json file which can be created with random data.

Required album fields:

  • title (string)
  • viewCount (number)
  • thumbnail (string) - Utilize the image placeholder service for dynamic image generation (https://dev.me/products/image-placeholder).
  • imageUrl (string) - Utilize the image placeholder service for dynamic image generation (https://dev.me/products/image-placeholder).

Album Details View

  • Display detailed information about the selected album.
  • Present viewCount in a summarized format (e.g., 1K, 1M, 1B, 1T, etc.).

Technical Requirements

  • CSS Framework: Choose a CSS framework (e.g., Material, Tailwind CSS) to ensure a responsive layout.
  • State Management: Implement NgRx for state management to manage application state efficiently.
  • Unit Testing: Write unit tests using Jest to ensure code quality and robustness.
  • Custom Components: Create custom pipes (e.g., for view count formatting), directives, and Angular routes.
  • UI/UX: Prioritize a clean, user-friendly interface with good design principles.
  • Organization: Structure the project code in a well-organized and maintainable manner.

Evaluation Criteria

  • NgRx for State Management: 10 points
  • Unit Tests Using Jest: 8 points
  • Pipes & Directives & Routes: 7 points
  • UI/UX: 5 points
  • Application Organization: 5 points

Submit your completed Angular project, including the albums.json file and any relevant documentation using git.
